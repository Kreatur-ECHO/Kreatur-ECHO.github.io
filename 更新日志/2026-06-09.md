# 更新日志 — 2026-06-09

> 项目第二天。从前一天晚上的原型快速迭代到完整博客系统，涵盖评论、点赞、管理面板、特效、SCF 后端。

## 🏗️ 基础设施

### GitHub Actions 工作流
- **`deploy.yml`**：推送 main 自动部署 GitHub Pages，注入 `REACTIONS_TOKEN`，刷新 `comments.json`
- **`fetch-comments.yml`**：每 10 分钟从 Issue #2 拉取评论到 `data/comments.json`，作为 API 回退
- 修复 cron 调度停滞问题

### SCF 云函数
- 创建 `blog-like` Web 函数（Node.js 18，监听 9000 端口，广州）
- **路由**：
  - `GET/POST /` — 点赞（IP MD5 去重，每 IP 每评论仅一次）
  - `GET/POST /visits` — 访问计数（每日 IP 去重）
- 使用 COS SDK 兼容的 HMAC-SHA1 签名算法写入 COS
- 零外部依赖（仅 Node.js 内置模块）

### COS 对象存储
- 创建 `blog-likes-1234567890-1441674200` bucket（广州，公有读私有写）
- 存储 `likes.json`、`visits.json`

## ✨ 新增功能

### 评论系统
- 浏览器直接拉取 GitHub Issues API（实时，零延迟）
- 本地 `data/comments.json` 作为 API 不可用时的回退
- **评论点赞**：前端 `localStorage` 去重 + 后端 SCF IP MD5 去重，双重保障
- 热门/最新排序切换
- 简易 Markdown 渲染（代码块、粗体、斜体、图片、链接）
- 复制评论、跳转 GitHub 查看原文

### 管理面板 (`#admin`)
- 通过 GitHub API 直接编辑/新增/删除文章
- 自动序列化 `posts.js`，提交时过滤空白文章
- 表单验证（title/content 非空）
- 发布/删除后自动刷新页面

### 文章系统
- 博客文章数据 `data/posts.js`（8 篇文章）
- 文章详情页（哈希路由 `#post/<id>`）
- 文章排序栏：最新 / 最多浏览，星标文章始终置顶
- **空白文章过滤**：渲染和保存时自动过滤
- 文章卡片统一紫色边框，flexbox 等高

### 访问计数
- SCF + COS 持久化存储，每日 IP 去重
- 前端显示累计访问数

### 特效
- Canvas 粒子系统（鼠标粒子 + 点击波纹）
- Hero 区域白色流星粒子
- 头像轨道发光点（紫色，缩放时跟随）
- 导航切换时防止粒子堆积

### UI/UX
- 星标文章徽章（⭐ 黄色，贴近角落，微发光）
- 文章卡片 hover 缩放 5% + 边框加粗（outline 防布局抖动）
- GitHub 统计静态回退值（API 限流时显示）
- QQ 社交按钮（hover 弹窗 + 一键复制 QQ 号）
- 头像羽化边缘内阴影
