## SCF 网易云音乐重构

### 背景
- `NETEASE_COOKIE` 环境变量中 `MUSIC_U` 过期，网易云 API 返回 `-447`
- 之前已部署自动登录续期功能（WeAPI），但 `rawCode` 只匹配 `20001`，漏掉 `-447`
- 扩大触发条件到所有非 200 码后，自动登录被触发但返回 `8821`（验证码拦截）
- 网易云 `MUSIC_U` Cookie 绑定 IP，用户浏览器提取的 Cookie 从 SCF/其他 IP 调用均返回 `20001`

### 根因
1. 「我喜欢的音乐」歌单（`2488546807`）是特殊歌单，API 强制要求登录
2. SCF 服务器 IP 触发网易云验证码，无法通过 WeAPI 手机号登录
3. Cookie IP 绑定导致从浏览器复制 Cookie 的方案也无效

### 方案：公开歌单
- 用户创建普通公开歌单「BLOG」（ID `18080794017`）
- 普通公开歌单 API 无需登录即可访问（参考：官方歌单 `3778678` 无需 Cookie 返回数据）

### SCF 代码变更 (`scf/app.js`)

| 操作 | 说明 |
|------|------|
| **删除** | WeAPI 加密模块（`aesEncrypt`/`rsaEncrypt`/`weapiEncrypt`，~60 行） |
| **删除** | 自动登录模块（`loadNeteaseAuth`/`saveNeteaseAuth`/`neteaseLogin`/~60 行） |
| **删除** | 环境变量引用 `NETEASE_UID`/`NETEASE_COOKIE` |
| **修改** | `fetchRecentSongs()` 去除 cookie 参数，歌单 ID 改为 `18080794017`，加 User-Agent + Referer 头 |
| **简化** | `handleRecentSong()` 移除认证/自动登录/回退逻辑，代码量减半 |

### 结果
- `/recent-song` 每次返回干净数据，无 `_diagnose`、无 `fromCache`
- 不再需要任何 NETEASE 环境变量
- 后续更新歌单：直接在网易云「BLOG」歌单增删歌曲，全自动同步（每天 4:00 刷新）

## 切歌双重播放修复 (`js/main.js`)

### 问题
快速切换歌曲时两首歌同时播放（叠加）

### 根因分析

| 问题 | 位置 | 说明 |
|------|------|------|
| 弹出列表无防抖 | `popupEl click` 监听器 | 旧代码无 `clickLock`，连点两首触发两次 `switchToSong` |
| 旧音频未彻底释放 | `switchToSong` 第 860 行 | 仅 `pause()` + `remove()`，音频流仍可能残留 |
| 竞态条件 | `switchToSong` 异步 fetch | 慢请求回来时可能覆盖快请求已创建的新音频 |

### 修复

1. **弹出列表防抖锁**：与黑胶点击一致，加 `clickLock`（500ms）
2. **强制释放旧音频**：`pause()` → `src = ''` → `load()` → `remove()` 四步彻底断开
3. **竞态丢弃**：新增 `_switchSeq` 序列号，fetch 返回后检查是否为最新调用，否则丢弃
