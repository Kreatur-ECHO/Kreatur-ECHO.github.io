## 下午/晚间 — 大量 UI 打磨

### 音乐播放状态保持
- 页面导航时保留当前播放状态，不再中断切换歌曲
- `loadRecentSong()` 首次调用才触发 `switchToSong()`，后续仅重建 UI
- 音频事件回调用 `getElementById` 动态查找碟片 DOM，避免闭包捕获旧引用

### 音柱可视化（最终方案：随机动画）
- 经历 Web Audio API → CORS 跨域问题 → `createMediaElementSource` 静音 → `captureStream` 无数据 → 最终放弃音频频谱
- **最终方案**：72 根音柱 5° 环绕黑胶，播放时 `setInterval` 70ms 随机跳动，暂停时 0.5s 渐出回落
- CSS `transition: height 0.5s ease-out` 平滑过渡
- 低频区（0-23）暖紫、中频区（24-47）亮紫、高频区（48-71）冷蓝
- 缩放偏移修复：`backface-visibility: hidden` + `will-change` GPU 合成

### 回顶按钮重设计
- 移入黑胶 wrapper 上方，共享同一图层
- `IntersectionObserver` 监听 Hero 区域控制显隐（替代不可靠的 scroll 事件）
- 水平居中于黑胶圆心 (`left: 50%; translateX(-50%)`)
- 桌面端 CSS 隐藏（侧边栏已有），手机/平板才显示
- z-index 1001 确保在最顶层

### 卡片布局大修
- 标题固定 2 行 clamp + `min-height`，消除长度差异
- 简介桌面端改用 `mask-image` 渐变遮罩渐隐（替代 `line-clamp` 硬截断）
- 手机端保持单行省略号
- 卡片固定 `height: 310px`，简介固定 `height: 120px`
- 标签 `margin-top: auto` 沉底
- 标签溢出检测：全量渲染 → 运行时 `scrollWidth > clientWidth` 检测 → 逐末位移除 + 追加 `...`

### 黑胶弹出列表
